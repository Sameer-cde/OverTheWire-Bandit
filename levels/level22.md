# Bandit Level 22 → Level 23

##  Objective
Find the password for `bandit23` which is hidden inside a dynamically named temporary file generated by an automated cron job script.

---

##  Commands Used
* `cat /etc/cron.d/cronjob_bandit23` — Read the cron configuration file.

* `cat /usr/bin/cronjob_bandit23.sh` — Inspected the automated bash script execution steps.

* `echo I am user bandit23 | md5sum` — Manually calculated the MD5 encryption hash string.

* `cat /tmp/<hash_string>` — Retrieved the final password from the temporary directory.

---

##  What I Learned

* **Cron Jobs:** I learned that Linux uses background schedulers (`cron.d`) to automate scripts at regular intervals (e.g., every minute).

* **Dynamic Paths:** Scripts don't always use simple text file names. They can use environmental variables like `$(whoami)` and cryptographic hashes to generate file paths on 
the fly.

* **MD5 Hashing:** `md5sum` takes a predictable string of input text and turns it into a unique, fixed-length fingerprint string. Re-running the exact same string allows you to predict the output filename.

---

##  Troubleshooting & Mistakes Along the Way
* **Wrong Level Script:** Initially, I logged into `bandit22` and accidentally read the `cronjob_bandit22.sh` script. I realized this script was for the *previous* level and contained the password I already used to log in. To progress, I had to look at the script 
targeting the next user: `cronjob_bandit23.sh`.

* **Path Awareness:** Making sure I was calculating the hash for user `bandit23` instead of my currently logged-in user context so the file path matched what the system root executed.
